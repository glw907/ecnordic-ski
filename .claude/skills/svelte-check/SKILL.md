Run `npm run check` from the project root and interpret the results.

## Steps

1. Run the check:

```bash
npm run check
```

This runs `svelte-check --tsconfig ./tsconfig.json` across all `.svelte`, `.ts`, and `.md` (mdsvex) files.

2. Interpret the output:

- **Clean:** output ends with `svelte-check found 0 errors and 0 warnings`. Report success and proceed.
- **Errors:** list each error with its file path and line number. Do not mark work complete until all errors are resolved.
- **Warnings only:** report them but don't block progress. Warnings are informational.

3. If there are errors, fix them before committing. Re-run after fixing to confirm clean.

## Notes

- `cloudflare:email` type errors may appear if `@cloudflare/workers-types` isn't installed or `tsconfig.json` doesn't include it under `"types"`. Fix: `npm install --save-dev @cloudflare/workers-types` and add `"types": ["@cloudflare/workers-types"]` to `compilerOptions`.
- mdsvex `.md` files used as routes (e.g. `+page.md`) are checked like Svelte components. `$props()` and imported components must be typed correctly.
- The `.svelte-kit/tsconfig.json` that `tsconfig.json` extends is generated by SvelteKit. If it's missing, run `npx svelte-kit sync` first.
